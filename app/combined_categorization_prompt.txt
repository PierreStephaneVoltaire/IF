## CATEGORY SCORES

### coding
Code implementation, refactoring, bug fixes (regardless of fix size), creating files, complex system implementations, Terraform modules, Kubernetes manifests, Helm charts, Dockerfiles, CI/CD pipeline definitions (GitHub Actions, GitLab CI, ArgoCD application configs), monitoring setup (Prometheus rules, Grafana dashboards), security implementations (OPA policies, network policies), AI/ML code, training scripts, data processing pipelines, automation scripts, backend code, API development. Also includes code review, explaining what a specific piece of code does, debugging a specific error in code, writing tests, and generating or modifying any config file. Any task where the expected output includes writing, reading, explaining, or modifying code or configuration files.

### architecture
System design, architecture planning, infrastructure design, debugging strategies, error handling patterns, resilience patterns, FinOps and cost optimization, capacity planning, SRE platform design, SLO/SLI/SLA planning, security architecture, threat modeling, AI/ML infrastructure planning, platform engineering, backend design, software design, roadmap planning, technical RFC review, ADR writing, architecture critique and evaluation. Also includes high-level technology choices, comparing frameworks or tools, scaling strategies, migration planning, and trade-off analysis between approaches. These are complex discussions requiring senior-level reflection. The output is plans, diagrams, trade-off analysis, or professional evaluations — not actual code.

### shell
Code or shell command requests that would result in one-liner shell commands or very short shell scripts (nothing beyond ~15 lines). The asks are usually how-to's or what's. They can also be questions related to CLI flags or options. Shell command suggestions, one-liners, CLI help, piped commands, kubectl, helm, terraform, docker, git, argocd, aws cli, az cli, gcloud, prometheus, openssl, security scanning tools, networking commands (curl, httpie, ssh, scp, dig, traceroute), performance tools (top, htop, strace, netstat, ss), jq/yq, awk, sed, grep, environment/path/permissions troubleshooting. If the request would result in a full script with logic, loops, or error handling beyond a few lines, prefer coding instead.

### social
Greetings, farewells, onomatopoeia, emotional one-word responses, casual chat, compliments, insults, small talk, reactions, acknowledgments, expressions, internet slang, meme speak, jokes, emotional support, banter, "thanks", "got it", "lol", "nice". Social interactions where there is no substantive information request. If the message contains any actual question or task embedded in casual framing, prefer the category that matches that task over social.

### financial
Only for queries where the primary need is financial instrument knowledge, market data, or structured financial analysis. Excludes any scenario where the core question is about a relationship, personal judgment, interpersonal conflict, or behavior that merely involves money as context — e.g. lending money to a friend, someone spending irresponsibly, back pay disputes, whether to cut someone off financially, or reacting to someone else's financial decisions. Those belong in general. Queries must be substantively about: ETFs, stocks, crypto, investment accounts, bonds, options, investing, trading, technical analysis, risk management, taxes, retirement, FIRE, saving, budgeting, debt management, investing strategies, financial planning, financial independence, financial education, financial news, financial advice, financial analysis, financial reporting, financial statements, financial ratios, financial metrics, financial modeling, financial forecasting, portfolio strategy, market commentary.

### health
Physical health, fitness, and nutrition. Diets, meal planning, macros, supplements, powerlifting, bodybuilding, workout programming, recovery, sleep, mobility, deload strategies, injury management, powerlifting competition preparation, peaking, water cutting, weight management, body composition, bloodwork interpretation. Also includes questions about symptoms, medications, mental health, medical conditions, and general wellness. Does not include questions about healthcare costs, insurance, or medical billing — those belong in general or financial.

### general
General questions or requests that don't fit into any other category. Always non-technical. Includes personal decisions or advice where money, numbers, or financial terms appear as context but the real question is about relationships, behavior, judgment, or values — these belong here, not in financial. Examples: should I lend money to a friend, my coworker got a raise and acts differently, someone spent their back pay on impractical things and is now asking for help. Also includes: general knowledge, things that would require web search, moral advice, rants, venting, opinions about what topics to learn, proofreading or rewriting non-code text, career advice, creative writing, brainstorming, philosophical questions, and any personal decisions that don't fit a more specific category.

## REASONING PATTERN SCORES

### simple
Factual questions requiring direct, straightforward answers. The answer is stable, well-defined, and doesn't require synthesis across multiple sources or perspectives. Examples: "what does this flag do", "what's the syntax for X", "what year did Y happen", "what's the difference between A and B" when the answer is unambiguous.

### opposing_perspective
Questions or decisions that benefit from considering two distinct, opposing viewpoints. Useful for debates, ethical dilemmas, controversial topics, or decisions where there are two clear camps. Examples: "should I use microservices or monolith", "is this approach good or bad", "pros and cons of X", "should I do A or B", evaluating trade-offs between exactly two options.

### multi_perspective
Design decisions, trade-offs, comparisons, or judgment calls that benefit from weighing multiple (more than two) viewpoints or approaches. Applies when there is no single correct answer and the quality of the response depends on exploring competing options, risks, or priorities. Examples: choosing between three or more architectures, evaluating a system design with many stakeholders, deciding on a technology stack with multiple viable options, reviewing a strategy from different angles.

### sequential_refinement
Tasks that involve producing, improving, or iterating on a concrete artifact — whether in one shot or across multiple turns. This includes writing or fixing code, generating scripts or configs, drafting or editing documents, implementing features, and reviewing or refactoring existing work. The key signal is that the output is a tangible artifact being built or improved.

### research
Queries where producing an accurate response requires access to current or rapidly changing data — where information from hours or days ago may differ meaningfully from last week. Applies to financial markets, stock prices, tax law changes, health guidelines, current events, and any domain where recency is critical to correctness. Does NOT apply to technical documentation, syntax, cloud service APIs, or developer tooling, as those change slowly enough that static knowledge suffices.
